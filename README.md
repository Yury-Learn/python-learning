## Как я запускаю скрипты

Пример запуска:

```bash
python3 week1_day1.py
```

Где:

* `python3` — интерпретатор Python 3.
* `week1_day1.py` — мой учебный скрипт.

Jupyter-ноутбуки (`.ipynb`) я открываю прямо в VS Code:

1. Открываю файл, например `week1_day2.ipynb`.
2. Выбираю интерпретатор Python (обычно виртуальное окружение `.venv`, если оно есть).
3. Запускаю ячейки по очереди и экспериментирую с кодом.

## Как я использую ChatGPT

Я не прошу «сделай за меня весь код».

Вместо этого я прошу ChatGPT:

* объяснить, что делает каждая строка моего кода;
* помочь понять ошибку и причину, а не просто дать готовый фикс;
* подсказать, как структурировать проект (имена файлов, функции, аргументы);
* предложить более «чистый» вариант решения, но так, чтобы я понимал, что именно поменялось.

Это важно для практики, потому что цель — не просто «код работает», а «я понимаю, что я делаю».

## Стиль кода в этом репозитории

Чтобы код выглядел аккуратно и привычно для других инженеров, придерживаюсь простых правил:

1. **Имена файлов**

   * Учебные дни: `week1_day1.py`, `week1_day2.ipynb` и т.п.
   * В будущем мини-утилиты: `log_parser.py`, `inventory_report.py` и т.д.
   * Только латиница и нижний регистр, без пробелов.

2. **Имена переменных**

   * Использую `snake_case`: `server_list`, `host_name`, `error_count`.
   * Название отражает смысл:

     * `servers` — список серверов;
     * `server` — один сервер (элемент списка);
     * `log_line` — одна строка лога.

3. **Имена функций**

   * Также `snake_case`, глагол + существительное:

     * `read_log_file()`
     * `parse_log_line()`
     * `print_inventory_table()`
   * Функция отвечает за одну конкретную задачу.

4. **Комментарии и язык**

   * Имёна переменных и функций — по-английски.
   * Комментарии и README — по-русски (так проще объяснять себе и другим).
   * Комментарии пишу, когда:

     * логика неочевидна;
     * есть привязка к специфике админской задачи (формат логов, особенности хостов и т.п.).

5. **Форматирование**

   * Отступы — 4 пробела.
   * Максимальная длина строки — стараюсь около 79–100 символов.
   * Одинаковый стиль кавычек в рамках одного файла (`"` или `'`).
